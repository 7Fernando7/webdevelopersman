HTML5 y CSS3

INTRODUCCIÓN

HTML5 es la actualización de HTML, el lenguaje en el que es creada la web. HTML5 también es un termino de marketing para agrupar las nuevas tecnologías de desarrollo de aplicaciones web: HTML5, CSS3 y nuevas capacidades de Javascript.

La versión anterior y más usada de HTML, HTML4, carece de características necesarias para la creación de aplicaciones modernas basadas en un navegador.

Flash en especial ha sido usado en reemplazo de HTML para desarrollar web apps que superan las habilidades de un navegador: Audio, video, webcams, micrófonos, datos binarios, animaciones vectoriales, componentes de interfaz complejos, entre muchas otras cosas. Ahora HTML5 es capaz de hacer esto sin necesidad de plugins y con una gran compatibilidad entre navegadores.

En HTML 5 usaremos como referencia una nueva estructura de elementos que cambiarán la manera de elaborar el contenido (sobre todo de blogs) y la manera en que buscadores lo analizan.

Nuevas etiquetas

<header> está diseñada para reemplazar las necesidad de crear divs sin significado semántico.

<hgroup> permite colocar un h1, h2, h3 dentro del header sin afectar el SEO, permitiendo usar otro h1 en el sitio.
En el HTML actual, sólo puedes usar h1 una sola vez por sitio o el h1 pierde prioridad de SEO.

<nav> Igual que<header>,<nav> está diseñado para que ahí coloques la botonera de navegación principal. Puedes colocar cualquier etiqueta dentro, aunque lo recomendado es usar listas <ul>

<section> Define un área de contenido única dentro del sitio. En un blog, sería la zona donde están todos los posts. En un video de youtube, habría un section para el video, uno para los datos del video, otro para la zona de comentarios.

Nuevas etiquetas II

<article> Define zonas únicas de contenido independiente. En el home de un blog, cada post sería un article..

<aside> Cualquier contenido que no esté relacionado con el objetivo primario de la página va en un aside. En un blog, obviamente el aside es la barra lateral de información.
En el home de un periódico, puede ser el área de indicadores económicos, bibliografía, fuentes...

<footer> Este es obvio. Es el pie de página y todo lo que lo compone.

Atención el<div>no está muerto:

Estas nuevas etiquetas no significan que ya no se use <div>. Div siempre debe usarse cuando necesites una caja con objetivos de diseño grafico o cualquier cosa que no tenga significado semántico. Sólo usa etiquetas semánticas de HTML5 donde sean necesarias.

Nuevas etiquetas III

Las etiquetas semánticas, a pesar de ser claves para posicionamento en buscadores y buen desarrollo web, no son la razón por la que todo el mundo habla de HTML5. Video, audio y animación vectorial están en la lista de prioridades y en la boca de todas las personas que evangelizan su uso.
Especificamente, las nuevas etiquetas son:

<video> Inserta video sin necesidad de plugins.

<audio> Lo mismo que video, pero sin video.

<input*> Input ya existía como la etiqueta para insertar cajas de texto y botones. Ahora es más poderosa.

<canvas> Un área de dibujo vectorial y de bitmaps con Javascript.

<svg> Una etiqueta, igual que<img>, para insertar dibujos y animaciones vectoriales al estilo de Flash.

Compatibilidad con navegadores

HTML5 trae al mundo del desarrollo web habilidades que, siendo el 2010, era casi vergonzoso no tener. Cosas como <video>,<canvas> eran más que necesarias. sin embargo, no es el primer gran cambio de internt.
La primera gran evolución del desarrollo web vino en el 2004 con Gmail, al traer a la luz el objeto XmlHttpRequest, más conocido como AJAX.

Lo que pocos saben es que AJAX fue creado por Microsoft para Internet Explorer 5 en 1999. Sí, el navegador responsable de detener la innovación más importante sin la cual, cosas como gmail, Facebook, Google Maps o quizás Twitter, habrían sido inviables.

Internet Explorer, claro, es también la rázon de la poca adopción de HTML5. sin embargo, hay formas de que las etiquetas semánticas de HTML5 y atributos de CSS3 funcione en IE.

Compatibilidad HTML5 en el Explorer

HTML5 Enabling Script: Permite usar las etiquetas semanticas dentro de IE6, 7 y 8 como si fueran divs normales, estilizables por CSS. Sin este script, las etiquetas son ignoradas en IE y es imposible agregarles diseños a ellas o cualquier elemento dentro de ellas.

Para utilizarlo debes incluir esta línea en el HEAD de tu documento.

<!--[if It IE 9]>

<script src="http://HTML5shim.googlecode.com/svn/trunk/HTML5.js">

</script>

<![endif]-->>

httpp://remysharp.com/2009/01/07/html5-enabling-script/

Compatibilidad con CSS3

Para muchos navegadores es necesario usar prefijos ( cada vez menos pues se actualizan muy rapido), en versiones mas recientes de Firefox, Chrome u Opera.

Además muchas alternativas que están basadas en estos sistemas.

Los prefijos que podemos usar son:

-moz- Para Firefox, -webkit- Para Chrome, y -o- Para Opera

Ejemplo:
-webkit-border-radius:12px;
-moz-border-radius:12px;
border-radius:12px;

Sin embargo, no te asustes demasiado si ves muchos cuadros rojos. Recuerda los scripts que mencioné arriba como HTML5 Enabling Script. Gracias a ellos, muchos de esos rojos se convierten en verdes.

Manifesto I

Una de las tendencias que están surgiendo actualmente es que Internet ya no le pertenece a las páginas web, sino que este lugar lo han venido tomando progresivamene las aplicaciones web. Una de las limitaciones de la dependencia de aplicaiones en Internet es que necesitamos estar conectados constantemente a una red para tener acceso a la misma. Por suerte HTML5 está dando los primeros pasos para que esta limitación desaparezca: El acceso a las aplicaciones web haciendo uso del Cache.

Para usar esta opción seguiremos los siguientes pasos

Paso1: Crear un archivo col la extension .appcache donde introduciremos los archivos que queremos que tengan acceso de forma off-line

Ejemplo: NombreArchivo.appcache

Manifesto II

Paso2: Introduciremos en ese documento (formato txt), la lista de archivos y sus rutas correspondientes.

CACHE MANIFEST

# versión 0.1

index.html

web.png

Estilos/estilo.css

img/foto1.jpg

....

Paso3: Por último debemos dediarle al documento html, que queramos Activar esta opción, eso lo haremos como en el seguiente ejemplo:

<html manifest="nombreArchivo.appcache">

Contenido editable

Ahora en html5, cualquier contenido puede ser editable, sin usar formularios (textarea o input) con la propriedad contenteditable="true", e incluso, se pueden agregar listas ordenadas (<ol>) o desordenadas (<ul>).

Un ejemplo:

<section contenteditable="true">

<h2>Titulo<h2>

<p>Texto</p>

<ul>

<li>Elemento</li>

<li>Elemento</li>

<li>Elemento</li>

</ul>

</section>

Syntaxis: <element contenteditable="true|false">

VIDEO

Hasta ahora, no ha habido un estándar para la proyección de un video/película en una página web.

Hoy en día, la mayoría de los vídeos se muestran a través de un plug-in (como Flash). Sin embargo, los navegadores diferentes pueden tener diferentes plug-ins.

HTML 5 define un nuevo elemento que especifica un metódo estándar para incrustar un vídeo / película en una página web: el elemento <video>.

<video width="320" height="240" controls="controls">

<source src"movie.mp4" type="video/mp4"/>

<source src"movie.ogg" type="video/ogg"/>

Tu navegador no es compatible para ver este video.

</video>

AUDIO

Al igual que los vides hasta ahora el audio se usaba a través de un plug-in (como Flash).

<audio controls="controls">

Nuevos atributos para formularios
(input)

HTML 5 viene con un montón de atributos nuevos para los formularios lo que nos ahorrara mucho trabajo para validación de los mismos.

Como no existe mejor modo de aprender que hacerlo uno mismo, vamos a ir escribiendo en una página de formulario los siguientes ejemplos e iremos comentando cada uno.

Search
(input)

<input type="search" autofocus placeholder="search"name="nombre">

En input type estamos declarando el tipo de input, luego con Autofocus significa que al cargar la página, se autoseleccionará este campo.

El placeholder es el texto que aparecerá en el campo inactivo, de modo que cuando lo seleccionemos pues se quedará en blanco y en name, le asignamos cualquier nombre al campo.

Number

<input type="number" min="10" max="80" step="10" placeholder="numero"name="nombre">

Como dijimos, input type declara el tipo de input, min declara el número mínimo de caracteres, mas el número máximo, placeholder y name ya lo habíamos dicho.

Range

<input type="range" min="10" max="80" placeholder="range"name="nombre"/>

Date

<input type="datetime"/>

Se puede cambiar datetime por date, datetime-local, time, week, o month

Tel, email, url

<input id="input" name="input" type="email/>

Se puede cambiar email por url

Requerido

Ahora los formularios permiten el atributo 'required' para especificar que un campo es requerido. Estp significa que los datos del formulario no serán enviados si el campo marcado como requerido está vacío. Puedes utilizar este atributo de dos formas:

<label for="name"> First name*: </label>

<input type="text" name="input1" required/>

o

<label for="name"> First name*: </label>

<input type="text" name="input1" required="required"/>

Enfoque automático de atributos

<input type="text" name="fname" autofocus="autofocus"/>

Height y Width

<input type="image" src="img_submit.gif alt="submit" width="48 height="48" />

Nota: La altura y la anchura de los atributos sólo se utilizan con <input type="image"/>

Step

El atributo especifica los intervalos de paso número legal de un elemento <input type="number" name="points" step="3"/>

Datalist

Muestra una lista predefinida cuando insertamos un texto.

<input list="browsers"/>

<datalist id="browsers"/>

<option value="Internet Explorer"/>

<option value="Firefox"/>

Expresiones regulares

Si deseas que en un campo de un formulario solo se puedan escribir solo un número máximo de caracterés, o solo mayúsculas, etc., esto lo puedes hacer mediante expresiones regulares utilizadas directamente en el código HTML:

<input type="text" name="username" id="username" placeholder="4 <> 10" pattern="[A-Za-z]{4,10}" required/>
